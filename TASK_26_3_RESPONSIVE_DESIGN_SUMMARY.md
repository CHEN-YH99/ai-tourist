### 3. 响应式设计原则应用

#### TailwindCSS 断点使用
- **sm (640px)**：小屏幕设备（平板竖屏）
- **md (768px)**：中等屏幕（平板横屏）
- **lg (1024px)**：大屏幕（桌面）

#### 响应式类名模式
```
- 基础样式：应用于所有屏幕
- sm:前缀：640px 及以上
- md:前缀：768px 及以上（侧边栏显示/隐藏分界点）
- lg:前缀：1024px 及以上
```

#### 移动优先设计
- 从最小屏幕开始设计
- 逐步增强到更大屏幕
- 确保所有功能在移动设备上可用

### 4. 触摸交互优化

#### 按钮和交互元素
- 最小触摸目标大小：44x44px（通过 py-2 sm:py-3 实现）
- 充足的间距：gap-2 sm:gap-3 sm:gap-4
- 清晰的视觉反馈：hover 和 active 状态

#### 导航交互
- 侧边栏滑入/滑出动画：duration-300
- 覆盖层点击关闭
- 菜单项点击自动关闭（移动端）

#### 表单交互
- 响应式输入框大小
- 清晰的标签和错误提示
- 移动端全宽按钮便于点击

### 5. 响应式排版

#### 字体大小响应
```
标题：
- text-2xl (mobile) → text-3xl (sm) → text-4xl (lg)

副标题：
- text-sm (mobile) → text-base (sm) → text-lg (lg)

正文：
- text-xs (mobile) → text-sm (sm) → text-base (lg)
```

#### 间距响应
```
内边距：
- px-4 (mobile) → px-6 (sm) → px-8 (lg)
- py-6 (mobile) → py-8 (sm) → py-12 (lg)

间隙：
- gap-2 (mobile) → gap-3 (sm) → gap-4 (lg)
```

### 6. 性能优化

#### 条件渲染
- 使用 `hidden sm:flex` 等类名避免不必要的 DOM 元素
- 响应式图片大小减少移动端加载

#### 动画优化
- 使用 CSS 过渡而非 JavaScript 动画
- 平滑的 duration-300 过渡
